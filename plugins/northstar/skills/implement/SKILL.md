# implement —— 令红变绿（手段自选，契约自钉）

audit 通过、测试已全红之后。本技能退化为"令红变绿"——撰写已上移，这里只补实现。

northstar 流程本身即 TDD 的强化形态：README → 测试 → 实现，测试在实现存在之前已是只读既成事实。**故无需引擎、无需外部 plugin**——"测试先于实现"由流程顺序结构性保证，不靠纪律对抗倾向。

## 手段按规模自选
README + 测试一旦定死，执行形态随改动规模临场决定；形态变，不变量不变：

| 改动规模 | 执行形态 |
|---|---|
| 小范围修改 | inline 直接实施 |
| 整个模块 | subagent |
| 更大 / 可并行 | team agents / workflow |

## 变更入口
- 纯实现的小改动（不触及契约）→ 直接 inline 令其符合既有测试。
- 触及契约的改动 → **先回 architect/module 改 README + 测试，再回此实现**，不在实现端绕过契约。

## 不变量（无论何种手段，恒成立）
1. **验收标准只读**：测试与评审发现对执行循环只读——**不得篡改、不得预判、不得软化**。唯一合法回应是「满足它」或「判定它错而回踢上游」。这一条同时罩住两种作弊：改测试迁就实现、压评审让任务通过。
2. **测试错 = 契约缺陷信号**：回踢 module / architect 重新生成，绝不在循环内静默改测试打补丁。**收敛阀**：同一红测试回踢达阈值（默认 3 次，对齐 CLAUDE.md §33a）仍不收敛 → BLOCKED 升级给人，附该缺口回踢历史（过程态出境）。
3. **过程态不入契约**：中间决策、试错进 git log / Linear / 记忆，去向交工作流，**绝不渗进 README**。
4. **退出条件**：不是"测试绿了"，而是 **"测试绿了 且 契约可被全新快照无损替换"**。

## 模块级退出门：契约合规（自出，不外包）
全绿后，退出前自审一刀：**实现是否恰好 = 模块 README 契约——有无少做、有无多做**。这是"测试只读"不变量的执行面，属本框架 dogma，不交外部 review。
（这是状态机的**模块级**终点；**项目级**终点是 architect 收口确认架构结构测试全绿，两级退出门对称。）
（代码质量 / 精简是通用工程卫生，非本技能职责——由使用者全局 CLAUDE.md 接 code-simplifier / code-review 等工具，在此门之后收尾。）

## 若选子代理（吸收自实战，不依赖其 plugin）
子代理无记忆——只看你给的输入。故：
- **输入仅 README + 测试**：零上下文子代理须能独立完成 task。不能 → 是契约没写全，回踢 module。严禁夹带指向对话历史的指令。
- **串行派发**：绝不并行派发**有共享可变状态、会冲突的** implementer；并行仅限 task 间**无共享可变状态、且并行收益明显**时（对应上表"可并行 → team/workflow"，并行粒度是无冲突的独立 task，不是同一片代码的多个 implementer）。子代理提问必先答完再令其动手。
- **显式指定模型**：按 task 复杂度分级（机械转录用最便宜、判断任务用标准、架构与最终评审用最强），显式指定——否则静默继承最贵模型。
- **过程态走文件**：task 输入、报告、diff 经文件路径交接，不粘进 controller 的 context。

## 退出
全绿、契约合规门通过、契约未被偷改 → 通过。回到模块循环，按 architect 项目 README 声明的**模块清单**取**下一个未完成模块**走 module→audit→implement；全部模块完成则触发 architect 收口（项目级退出门）。architect 只在边界错（回踢）或收口时重入，**不逐模块派发**。
