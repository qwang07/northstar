# architect —— 系统拓扑设计

架构层、可重入。在任何模块详设之前；也在 module/audit 发现边界错时被回踢重入。产出项目根 README + 架构结构测试。

## 硬门
未与用户确认模块边界与依赖拓扑前，不进入 module、不写任何实现代码。产物未过 audit 不交棒。

## 入口判定（按模块粒度，不是项目一刀切）
逐模块问：该模块有无现实态代码？
- **无** → 直接探讨需求，凭空声明目标态拓扑。
- **有** → 先蒸馏：从 as-is 代码反推隐含 to-be，滤除过程态遗留（死路径、历史补丁、当年妥协），**并询问用户是否需要调整**。
- 半成品 = 两者按模块混合，天然处理。

## 蒸馏铁律
**代码是证据，不是答案。** 绝不把"现在怎么做的"直接抄成 README——那是让现实态僭越成目标态，SSOT 出生即被污染。产出必须是人机共同判断后的目标态。

## 对话方法论
- 一次一问，优先多选；聚焦目的 / 约束 / 成功标准。
- 大问题先分解为子模块（各自独立、关系清晰、构建顺序），再逐模块走流程。
- 每个模块须能回答：它做什么、它依赖什么、它服务谁/输出什么。不读内部能否理解它？能改内部而不破坏调用方吗？不能 → 边界没划好。

## 产出（守边界一：只边界不内部）
框架式项目 README，**只声明**：
1. **模块清单**与各自边界接口；
2. **模块间依赖拓扑**——每条边写成"A→B 经接口契约 I"，**边携带接口契约 I**，不只是"A 依赖 B"；
3. 刻意**不写模块内部**——即使探索时看到大量内部实现，忍住不写进去，那是 module 的领地。

项目 README = 接口索引 + 拓扑。接口契约 I 在此声明一次（接缝由此闭合：上游输出与下游输入日后都针对同一 I 测）。

## 架构结构测试（纯结构·工具无关）
派生测试，校验**声明的模块作为单元存在 + 各自声明的公共接口（接口契约 I 的形状）真实可见/可导入**。只需语言运行时/文件系统，**工具无关**。

- **不验真实依赖图、不验运行时接线**：边的语义兼容已由接缝接口契约 I 在上下游行为测试中双向锁住（再验"边存在"部分冗余，且验真实图需引入外部代码分析、破工具无关）；"A 真在运行时调用 B"（≠ 各自对 stub 兼容）属集成层，YAGNI 默认不做，真在意时由 README 显式声明系统级意图、落成集成性行为测试。
- **生命周期**：派生时全红（实现尚不存在）→ 随各模块 implement 完成而逐段变绿 → 全模块完成后由本 skill 收口（见退出）。
- **只测结构，不碰任何模块行为**。

## 守时间语义分层
拓扑契约里不得出现带时间戳的叙事——决策记录 / 变更说明 / 历史背景一律出境，路由交工作流。承重约束在热文档留一 token 标记（`[承重·勿删]`），理由本身出境。
**试金石：本 README 能否被一份全新快照无损替换？** 不能 → 历史已渗入，删掉。

## 退出（两相）
- **设计退出**：模块边界、依赖拓扑、接口契约清晰且经用户确认 → 交 audit 审。审过 → **项目 README 的模块清单即模块循环的迭代源**，逐模块交 module（module→audit→implement）；architect 不逐模块重入，只在回踢或收口时重入。
- **项目级退出门（收口）**：全部模块经 module→implement 完成后，本 skill 收口——确认**架构结构测试全绿**（声明的模块都到位、接口都可见）即**项目完成**。这是 loop 的终态，与 implement 的模块级退出门对称。派生者拥有收口。

## 回路收敛阀
本 skill 被 module/audit 反复回踢**同一缺口**达阈值（默认 3 次，对齐 CLAUDE.md §33a）仍不收敛 → **BLOCKED 升级给人决策**，附该缺口回踢历史（前几次各是什么缺口、怎么改的；过程态出境，进 git/Linear），不再自动重入。反复回踢同一条边常意味根因在更上游（边界本身划错），需人判断"再试"还是"上移重划"。
